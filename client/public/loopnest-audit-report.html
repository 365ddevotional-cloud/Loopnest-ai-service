4. Backend Logic

None. The entire game system is 100% client-side.

  • Zero API endpoints in server/routes.ts for games, LoopNest, or interactive features.
  • Zero database tables in shared/schema.ts for game data.
  • No server-side game state, scores, or user-game associations.

All game content is hardcoded in GameRegistry.ts. Draft saving uses localStorage key "gameBuilderDraft". Plan tier uses localStorage key "loopnest_plan".

5. Firebase / Database Connections

Firebase (LoopNest only)

  • client/src/loopnest/firebase.ts initializes a Firebase app using 4 environment variables.
  • Uses Firebase Authentication only — specifically getAuth().
  • Auth methods used: signInWithEmailAndPassword, createUserWithEmailAndPassword, sendEmailVerification, onAuthStateChanged, signOut.
  • No Firestore, no Realtime Database, no Firebase Storage, no Cloud Functions.

PostgreSQL

The game module has zero connection to the project's PostgreSQL database. No schema tables, no queries, no ORM usage.

9. Admin-Only Controls

None exist for game creation. The admin dashboard (/admin) has zero references to games, LoopNest, or interactive content.

  • The CreateGamePage at /interactive/create is publicly accessible — anyone can use it without authentication. It creates temporary in-browser games using localStorage drafts and base64-encoded share links. No server persistence.
  • The LoopNest builder at /loopnest/builder requires Firebase authentication but simply wraps the same CreateGamePage component with an auth guard and a "Back to Dashboard" button.

10. Standalone Extraction Feasibility

Verdict: Safe to Extract

The LoopNest/game module can be cleanly separated from the 365 Daily Devotional app with minimal effort and zero functional breakage to the devotional app.

Dependencies on the 365 App (things to sever)

DependencyFileWhat It ImportsSeverity
ThemeContext GamePage.tsx, GamesHub.tsx, CreateGamePage.tsx useTheme() → resolvedTheme EASY
Wouter routing All pages useLocation, useRoute, Link EASY

Home page link Home.tsx line 76 Link to /interactive/tap-the-promise EASY
GameConsoleIcon Header.tsx line 4 SVG icon import EASY

What LoopNest/Games Do NOT Depend On

  • No PostgreSQL / Drizzle ORM
  • No Express server / API routes
  • No TanStack Query
  • No shared schema types
  • No devotional data, Bible data, or Sunday School data
  • No offlineDb / IndexedDB sync
  • No notification system
  • No service worker logic
  • No admin auth system

Extraction Steps

  1. Copy client/src/loopnest/, client/src/game-engine/, client/src/interactive/ into a new project
  2. Copy client/public/audio/tap-theme.mp3
  3. Extract the 113 .ttp-* CSS lines from index.css into a standalone file
  4. Replace useTheme() with a local theme provider or hardcoded theme
  5. Remove 2 links from Header.tsx and 1 from Home.tsx in the 365 app
  6. Remove LoopNest route registrations (lines 85–91) and imports (lines 49–54) from App.tsx
  7. Remove the LoopNestAuthProvider wrapper logic (lines 110–115) from App.tsx

Impact on 365 App After Extraction

Zero functional breakage. The devotional app, Bible reader, Sunday School, prayer system, offline sync, and admin dashboard have zero code dependencies on the game module. The only visible change would be the removal of the game controller icon in the sidebar nav and the game link on the homepage.
